## data 模块说明

`data` 目录用于存放项目所需的各类数据文件，包括原始数据集、预处理结果以及医疗领域词典等。

- **总体职责**
  - 集中管理训练数据、评估数据和运行时依赖的数据文件；
  - 清晰区分“原始数据”和“处理后数据”；
  - 为知识图谱构建、向量库构建等步骤提供数据源。

---

### raw 子目录

- **路径**：`data/raw/`
- **作用**：存放“原始 / 半结构化”的数据文件，一般不直接修改。
- **典型文件**
  - `data.jsonl`、`train.jsonl`、`dev.jsonl`、`dialog.jsonl`
    - 可能包含问答样本、对话数据、训练/验证集等；
    - 可用于微调、评估或构建向量索引。
  - `medical.json`
    - 可能为结构化的医疗知识数据，用于构建知识图谱或训练语料。

> 建议：对 `raw` 中的文件尽量只读，所有清洗、转换步骤在脚本中完成，将结果写入 `processed`。

---

### processed 子目录

- **路径**：`data/processed/`
- **作用**：存放从 `raw` 清洗、转换后的结果，例如：
  - 已分词 / 标注过的数据；
  - 适用于训练 / 导入 Milvus 的干净文本；
  - 构建知识图谱时中间产物等。

> 目前仓库中该目录为空，可按项目需要补充：  
> 建议所有数据处理脚本都把结果输出到 `processed/`，同时在文档中说明生成方式。

---

### dict 子目录

- **路径**：`data/dict/`
- **作用**：存放与医疗领域相关的词典 / 词表文件，用于：
  - 知识图谱构建（疾病、症状、药品、科室等实体）；
  - 规则匹配、实体识别辅助；
  - 构造正则 / 关键字列表等。

- **典型文件说明**
  - `check.txt`：检查项目名称（如血常规、CT、核磁等）
  - `deny.txt`：否定词（如“不是”、“无”、“未见明显”等）
  - `department.txt`：科室名称（如内科、外科、心内科等）
  - `disease.txt`：疾病名称词表
  - `drug.txt`：药物名称词表
  - `food.txt`：食物名称词表（用于饮食相关知识图谱）
  - `producer.txt`：药品生产企业名称
  - `symptom.txt`：症状名称词表

这些词典在构建 Neo4j 知识图谱、实现规则增强（如基于关键词的意图识别）时非常有用。

---

### 使用建议

- 数据处理脚本（如构建向量库、导入图谱等）应在文档中说明：
  - 输入来自 `data/raw/` 中哪些文件；
  - 输出写入 `data/processed/` 或 `storage/` 中的哪些文件；
  - 是否可以重复运行、是否幂等。
- 如有数据更新（新增疾病、药品等），建议：
  - 先更新 `dict/` 中对应词典；
  - 然后重新跑一次构建脚本，更新 Milvus / Neo4j 中的数据。
